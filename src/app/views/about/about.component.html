Sobre o projeto

O projeto em Angular é uma calculadora de empréstimo com juros simples que permite aos usuários simular empréstimos para diferentes finalidades, como casa, carro e pessoal. Ele apresenta uma interface atraente e responsiva, com um design agradável e fácil de usar.

O projeto utiliza técnicas avançadas de desenvolvimento Angular para criar uma experiência de usuário interativa e informativa.

Descrição Técnica

O projeto é uma aplicação Angular que segue as melhores práticas de desenvolvimento e utiliza diversas técnicas avançadas. Aqui estão os principais componentes e técnicas utilizadas:

  1. Módulos e Componentes Angular:

    • `AppModule`: O módulo raiz do aplicativo Angular.
    • `AppRoutingModule`: Módulo de roteamento que gerencia as rotas do aplicativo.
    • `AppComponent`: O componente raiz do aplicativo.
    • VIEWS:
      • `HomeComponent`: Componente da página inicial localizado na pasta `views/home`.
      • `AboutComponent`: Componente da página "Sobre" localizado na pasta `views/about`.
      • `LoanCalculatorComponent`: Componente da calculadora de empréstimos localizado na pasta `views/loanCalculator`.
    • COMPONENTS:
      • `NavbarComponent`: Componente da barra de navegação localizado na pasta `components/navbar`.
      • `FooterComponent`: Componente do rodapé localizado na pasta `components/footer`.
      • `GraphBarComponent`: Componente que exibe gráficos de barras localizado na pasta `components/graphBar`.
    • MODEL:
      • `Graph`: Uma interface que define a estrutura de dados usada para criar gráficos de barras.
    • SERVICES:
      • `LoanCalculatorService`: Um serviço que lida com a simulação de empréstimos. Ele faz chamadas HTTP para um serviço de simulação.
    • db.json:
      • Um arquivo JSON que pode conter dados simulados ou informações do aplicativo, utilizado pelo serviço `LoanCalculatorService`.

  2. Roteamento:

    • O roteamento é configurado no `AppRoutingModule`, que define as rotas para diferentes componentes.

  3. HTML e CSS dos Componentes:

    • Cada componente possui seus próprios arquivos HTML e CSS para definir a estrutura e o estilo.
    • Os estilos estão definidos principalmente usando classes CSS.

  4. Serviços:

    • `LoanCalculatorService`: Um serviço que lida com a simulação de empréstimos. Ele faz chamadas HTTP para um serviço de simulação.

  5. Modelo de Dados:

    • `Graph`: Uma interface que define a estrutura de dados usada para criar gráficos de barras.

  6. Formulários e Validação:

    • Formulários são definidos nos componentes usando o módulo `FormBuilder`.
    • A validação de entrada é realizada usando as funcionalidades de validação do Angular.

  7. Roteamento de Navegação:

    • A navegação entre as páginas é feita usando diretivas `routerLink` nos links do componente de barra de navegação.

  8. Estilização:

    • A estilização é feita principalmente com CSS, com definições de estilos para os componentes e elementos da interface.

  9. Comunicação de Dados:

    • O componente `LoanCalculatorComponent` se comunica com o serviço `LoanCalculatorService` para realizar a simulação de empréstimo e atualizar os dados exibidos.

  10. Gráficos de Barras:

    • O componente `GraphBarComponent` é responsável por exibir gráficos de barras com dados de simulação.

  11. Responsividade:

    • Há regras de estilo definidas para tornar o aplicativo responsivo em telas menores.

  12. Requisições HTTP:

    • O aplicativo faz chamadas HTTP para um serviço simulado para obter resultados de simulação de empréstimo.
